# MainMenuWin.java - Документация

## Summary
Абстрактный класс `MainMenuWin` предоставляет базовую функциональность для работы с окнами, открываемыми через главное меню АРМ. Он стандартизирует процесс открытия окон через меню с настройкой таймаутов и обработкой исключений. Класс автоматически управляет таймаутами ожидания окон и предоставляет механизм повторных попыток при возникновении ошибок активации.

## Public методы
- **`openWindow(boolean fromMainMenu)`** - открывает окно через главное меню или напрямую (основной метод для использования в конструкторах)
- **`win()`** - возвращает объект Window (реализация интерфейса FormWin)
- **`beforeSearchingForWindow()`** - защищенный метод для переопределения логики между проходом по меню и поиском окна
- **`setTimeoutViaMainMenu()`** - устанавливает сокращенный таймаут для окон, открываемых через меню
- **`setDefaultTimeout()`** - восстанавливает стандартный таймаут окон

## Dependencies
- **`base.MainClass`** - основной класс фреймворка
- **`bugbusters.modules.helper.Helper`** - утилиты для повторных попыток
- **`com.rational.test.ft.WindowActivateFailedException`** - исключения RFT
- **`enums.FindBy`** - перечисление стратегий поиска
- **`env.Environment`** - доступ к настройкам окружения
- **`forms.general.abstractForms.FormWin`** - интерфейс окон форм
- **`userInterface.Window`** - класс окон UI

## Usage Example
```java
public class Шлюзы_Objects extends MainMenuWin {
    public Шлюзы_Objects(boolean fromMainMenu) {
        super("Справочники->ПВВС->Шлюзы", 
              FindBy.Class, "asdco.ARM_KO.FrmGate", 
              FindBy.CaptionText, "Шлюзы");
        openWindow(fromMainMenu);
        // инициализация элементов окна
    }
}
```

## Uncertainties
1. **Логика таймаутов** - жесткие значения 5 попыток и зависимость от конфигурационных свойств
2. **Обработка исключений** - автоматическое завершение АРМ при `WindowActivateFailedException` может быть избыточным
3. **Наследование от MainClass** - потенциальное наследование ненужной функциональности
4. **Инициализация полей** - конструктор требует передачи всех параметров поиска, что может быть негибким
