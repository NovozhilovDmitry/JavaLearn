# NewTXT.java - Документация

## Summary
Класс `NewTXT` предоставляет функциональность для работы с текстовыми файлами в тестовом фреймворке. Он поддерживает создание, поиск, сравнение и выгрузку текстовых файлов с различными кодировками и разделителями. Класс интегрирован с системой шлюзов и включает расширенные возможности для поиска и валидации содержимого файлов.

## Public методы
- **`NewTXT(Шлюзы FilePath, String FileName, Расширения_файлов расширение, Разделитель разделитель)`** - конструктор для создания файла
- **`NewTXT(String fileNamePart, Расширения_файлов расширение, Разделитель разделитель, Шлюзы... шлюзы)`** - конструктор для поиска существующего файла
- **`задатьКодировку(Кодировки кодировка)`** - устанавливает кодировку файла
- **`сравнить_содержимое_файлов(Object[][] obj)`** - сравнивает содержимое файла с ожидаемыми данными
- **`найтиПодстрокуВФайле(String искомая_подстрока)`** - ищет подстроку в файле
- **`найтиВсеЗаданныеПодстрокиВФайле(String[] искомые_подстроки)`** - ищет несколько подстрок в файле
- **`выгрузить_файл(Object[][] obj)`** - создает и выгружает файл с данными

## Dependencies
- **`enums.Кодировки`**, **`enums.Расширения_файлов`**, **`enums.Шлюзы`** - перечисления для настройки файлов
- **`utils.FileSearcher`** - утилита для поиска файлов
- **`base.MainClass`** - основной класс фреймворка
- **`io.qameta.allure.Step`** - интеграция с Allure
- **Внешние**: `java.io.*`, `java.nio.file.*`, `java.nio.charset.*`, `org.slf4j.Logger`

## Usage Example
```java
NewTXT txt = new NewTXT(Шлюзы.ШЛЮЗ_ПИС, "data", Расширения_файлов.TXT, NewTXT.Разделитель.Табуляция);
Object[][] data = {
    {"Поле1", "Поле2"},
    {"Значение1", "Значение2"}
};
txt.выгрузить_файл(data);
txt.найтиПодстрокуВФайле("Значение1");
```

## Uncertainties
1. **Разделители** - поддержка только табуляции, отсутствие других распространенных разделителей
2. **Производительность** - загрузка всего файла в память может быть проблемой для больших файлов
3. **Кодировки по умолчанию** - Windows-1251 жестко закодирована в конструкторе поиска
4. **Обработка исключений** - некоторые ошибки завершают тест, что может быть избыточно
