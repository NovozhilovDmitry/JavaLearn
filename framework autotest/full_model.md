# Gate.java

## Summary
Класс, представляющий шлюз - каталог для обмена данными с системой ДКО. Инкапсулирует информацию о пути к шлюзу и предоставляет методы для работы с его содержимым. Поддерживает получение списка файлов, проверку наличия файлов и форматированное строковое представление.

## Public методы
1. **Gate(String label, String absPath)**  
   Конструктор - создает шлюз с меткой и абсолютным путем.

2. **label()**  
   Возвращает метку шлюза (например: "tmp", "KCOI").

3. **path()**  
   Возвращает абсолютный путь к шлюзу как Path.

4. **children()**  
   Возвращает отсортированный список всех вложенных файлов и директорий.

5. **files()**  
   Возвращает только файлы (без директорий) в шлюзе.

6. **containsFiles()**  
   Проверяет наличие файлов в шлюзе (игнорируя подкаталоги).

7. **toString()**  
   Возвращает форматированное строковое представление шлюза с его содержимым.

## Dependencies
- java.io.File
- java.nio.file.Path
- java.util.List, Arrays, Collections

## Usage Example
```java
Gate gate = new Gate("ODBR", "A:\\credit\\run\\gate\\ODBR\\out\\");

// Проверка наличия файлов
if (gate.containsFiles()) {
    // Получение списка файлов
    List<File> files = gate.files();
    
    // Вывод информации о шлюзе
    System.out.println(gate.toString());
}
```

## Uncertainties
1. Нет обработки случаев, когда путь не существует или недоступен
2. Метод children() может вернуть пустой список по разным причинам
3. Сортировка в toString() по isFile может быть неочевидной
4. Не предусмотрены методы для фильтрации файлов по расширению или другим критериям

# Operation.java

## Summary
Класс, представляющий операцию как комбинацию действия (Runnable) с текстовым описанием и приоритетом. Уникальность операции определяется её описанием. Используется для организации выполнения действий в определенном порядке на основе приоритета. Рекомендуется использовать в контейнере Operations.

## Public методы
1. **Operation(String desc, Runnable op, int priority)**  
   Конструктор - создает операцию с описанием, действием и приоритетом.

2. **Operation(String desc, Runnable op)**  
   Конструктор - создает операцию с приоритетом по умолчанию (0).

3. **equals(Object o)**  
   Сравнивает операции по описанию.

4. **hashCode()**  
   Возвращает хэш-код на основе описания.

## Dependencies
- java.util.Objects (для equals и hashCode)

## Usage Example
```java
Operation operation = new Operation(
    "Создание пользователя", 
    () -> создатьПользователя("Иван"), 
    1
);

Operation simpleOp = new Operation(
    "Логирование", 
    () -> log.info("Тест завершен")
);
```

## Uncertainties
1. Нет валидации параметров конструктора (null проверки)
2. Приоритет может быть любым целым числом, нет ограничений
3. Не ясно, как обрабатываются исключения в Runnable действиях

# Operations.java

## Summary
Контейнер для хранения операций, расширяющий HashSet<Operation>. Отличается от стандартного Set тем, что при итерации операции сортируются по приоритету в порядке возрастания. Обеспечивает выполнение операций в порядке их приоритета.

## Public методы
1. **forEach(Consumer<? super Operation> action)**  
   Переопределенный метод для итерации по операциям, отсортированным по приоритету.

## Dependencies
- model.Operation
- java.util.HashSet
- java.util.function.Consumer

## Usage Example
```java
Operations operations = new Operations();
operations.add(new Operation("Операция 1", () -> {}, 2));
operations.add(new Operation("Операция 2", () -> {}, 1));

// Операции выполнятся в порядке: Операция 2, затем Операция 1
operations.forEach(op -> op.action.run());
```

## Uncertainties
1. Наследование от HashSet может быть не оптимальным для частых сортировок
2. Нет методов для массового добавления операций
3. Не предусмотрена обработка дубликатов по описанию (используется HashSet логика)
