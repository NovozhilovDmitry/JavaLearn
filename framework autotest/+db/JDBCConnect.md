# JDBCConnect.java

## Summary
Класс для управления соединениями с Oracle базой данных. Реализует пул соединений с синхронизацией для многопоточного доступа. Автоматически загружает драйвер Oracle и настраивает параметры соединения из свойств окружения. Обеспечивает единственное соединение на схему.

## Public методы
1. **get()**  
   Статический метод для получения соединения из пула. Возвращает Connection для текущего проекта.

2. **JDBCConnect()**  
   Конструктор по умолчанию - создает новое соединение.

3. **JDBCConnect(Connection existingConnection)**  
   Конструктор с существующим соединением.

4. **getConnection()**  
   Возвращает текущее соединение (не рекомендуется для прямого использования).

## Dependencies
- Oracle JDBC Driver (oracle.jdbc.driver.OracleDriver)
- Projects enum и Projects.DB_SCHEMAS для маппинга схем
- Environment.getDbProperties для параметров подключения
- JUnit 5 (Assertions.fail) для обработки ошибок

## Usage Example
```java
// Получение соединения через пул
Connection conn = JDBCConnect.get();

// Использование в Table классе
Statement stmt = JDBCConnect.get().createStatement();
```

## Uncertainties
1. Пул соединений не ограничен по размеру
2. Нет механизма проверки валидности соединений в пуле
3. Закрытие старых соединений в отдельном потоке может вызвать race conditions
4. Не обрабатываются случаи, когда соединение становится невалидным
5. AutoCommit отключен, что может привести к накоплению транзакций
6. Схема определяется через currentProject, что может быть небезопасно в многопоточности
