# Журнал_планов_досрочного_погашения.java

## Описание
Класс для работы с журналом планов досрочного погашения кредитов в системе АС ДКО. Предоставляет функциональность для создания новых записей о планах погашения, обработки существующих планов и проверки статусов обработки документов ED408. Включает вложенный класс для ввода новых записей с валидацией данных.

## Public методы
- `Журнал_планов_досрочного_погашения()` - конструктор по умолчанию
- `Журнал_планов_досрочного_погашения(boolean fromMainMenu)` - конструктор с опцией открытия из главного меню
- `getInstance()` - возвращает экземпляр класса (Singleton)
- `создать()` - нажимает кнопку "Создать" для добавления новой записи
- `обработка_планов(String номер_кредита, String долг)` - запускает обработку планов с проверкой сообщения о долге
- `обработка_планов()` - запускает обработку планов без параметров
- `проверка_статуса_и_номера__с_датой_обработки()` - проверяет статус и номер с датой обработки ED408
- `проверка_информационного_поля_Номер_заявления_КО(String)` - проверяет поле номера заявления КО
- `проверка_информационного_поля_Дата_заявления_КО(String)` - проверяет поле даты заявления КО
- `проверка_информационного_поля_Статус_обработки_ED408_этапа_контроля(String)` - проверяет статус этапа контроля
- `проверка_информационного_поля_Номер_и_дата_ED408_этапа_контроля(String, String)` - проверяет номер и дату этапа контроля
- `проверка_информационного_поля_Номер_и_дата_ED408_этапа_обработки(String)` - проверяет номер и дату этапа обработки
- `проверка_информационного_поля_Статус_обработки_ED408_этапа_обработки(String)` - проверяет статус этапа обработки

## Вложенный класс Ввод_новой_записи
- `заполнение_полей_и_сохранение(String номерПлана, String номер_кредита_кПогашению, String планируемая_датаПогашения, String реквизитыКО, boolean инициатор_погашения, String осн_долг_кПогашению, String сумма_проц_кПогашению)` - заполняет все поля формы и сохраняет запись

## Неясности
1. Разница между этапами "контроля" и "обработки" ED408 - бизнес-логика разделения
2. Назначение поля "инициатор погашения КО" - как влияет на процесс погашения
3. Форматирование суммы долга - почему используется замена запятых на точки и форматирование через BigDecimal
4. Логика проверки даты и времени в поле "№ и Дата ED408 этапа обработки" - почему время может присутствовать или отсутствовать
5. Назначение устаревших методов проверки полей - какие сценарии требуют их использования вместо новых методов
