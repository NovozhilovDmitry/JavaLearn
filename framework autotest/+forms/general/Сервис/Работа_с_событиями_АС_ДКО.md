# Работа_с_событиями_АС_ДКО.java

## Описание
Класс для автоматизации работы с событиями АС ДКО, в частности для закрытия операционного дня и связанных операций. Содержит вложенные классы для работы с диалоговыми окнами выбора параметров перехода, заключительных тестов и смены даты операционного дня. Интегрирован с базой данных для проверки календаря операционных дней.

## Public методы
- `Работа_с_событиями_АС_ДКО(boolean fromMainMenu)` - конструктор с опцией открытия из главного меню
- `getInstance()` - возвращает экземпляр класса (Singleton)
- `закрытие_опер_дня()` - нажимает кнопку закрытия операционного дня
- `закрыть()` - закрывает основное окно

## Вложенные классы

### Выберите_параметры_перехода
- `ввести_дату(String дата)` - вводит дату следующего операционного дня
- `выбрать_время_создания_длинной_точки(String значение)` - выбирает время создания точки
- `нажать_сохранить()` - нажимает кнопку сохранения параметров
- `выбрать_и_сохранить(String дата)` - устаревший комплексный метод

### Заключительные_тесты
- `продолжить()` - нажимает кнопку продолжения без проблем
- `продолжить(Boolean проблемы, int time_limit)` - устаревший метод с обработкой проблем
- `проверить_значения_строк(String... args)` - проверяет значения в таблице

### Смена_даты_операционного_дня
- Конструкторы с параметрами времени ожидания и типа информационного окна
- `проверка_текста_диалогового_окна_Сообщение(String text)` - проверяет текст сообщения

### ПереблокировкаОбеспеченияНаДату
- `закрыть()` - закрывает окно переблокировки
- `таблица()` - возвращает таблицу
- `getProgressBar()` - возвращает прогресс-бар

## Зависимости
- `base.MainMenuWin` - базовый класс окон главного меню
- `forms.general.dialog.Диалоговое_окно_с_действием` - диалоговые окна с действиями
- `forms.general.dialog.Сообщение` - окна сообщений
- `db.StatementChecker` - выполнение SQL-запросов

## Пример использования
```java
Работа_с_событиями_АС_ДКО работа = new Работа_с_событиями_АС_ДКО(true);
работа.закрытие_опер_дня()
      .ввести_дату("01.01.2024")
      .выбрать_время_создания_длинной_точки("не создавать")
      .нажать_сохранить()
      .да();
```

## Неясности
1. Назначение параметра `fromMainMenu` - как именно влияет на открытие окна
2. Логика определения "праздничного дня" и "выходного дня с признаком ОД ПС" - разница в обработке
3. Назначение системного свойства `Заключительные_тесты.init.timeout.millis` - конфигурирование таймаута
4. Разное поведение для проекта DEPOSIT - почему меняется следующее окно
5. Алгоритм ожидания окна "Произошла смена операционного дня" - цикл с попытками инициализации
