# Протокол.java

## Описание
Класс для работы с диалоговыми окнами протоколов и сообщений в системе. Поддерживает различные типы протоколов через перечисление, предоставляет функциональность для проверки сообщений, извлечения текста и таблиц из протокола. Реализует интерфейсы для стандартизации работы с диалоговыми окнами.

## Public методы
- `Протокол(String названиеОкна)` - конструктор с названием окна
- `Протокол(ОкноСообщения названиеОкна)` - конструктор с типом окна сообщения
- `getInstance()` - возвращает экземпляр класса (Singleton)
- `проверка_сообщения_по_части_строки(String)` - проверяет наличие части текста в сообщении
- `проверка_сообщения_БезТегов_по_части_строки(String)` - проверяет сообщение без HTML-тегов
- `проверка_отсутствия_сообщения_по_части_строки(String)` - проверяет отсутствие текста в сообщении
- `сохранение()` - нажимает кнопку "Сохранить" с возвратом текущего объекта
- `продолжить()` - нажимает кнопку "Продолжить" с возвратом текущего объекта
- `getText()`, `getText(boolean)` - возвращают текст сообщения
- `readTables()` - парсит текст окна в список таблиц

## Зависимости
- `forms.general.dialog.abst.DialogWithMessage` - абстрактный класс диалоговых окон
- `utils.HTMLtoCyrillic` - конвертер HTML символов
- `utils.XML.NewXML` - работа с XML таблицами
- `org.jsoup.Jsoup` - парсинг HTML

## Пример использования
```java
Протокол протокол = new Протокол(Протокол.ОкноСообщения.Результат_обработки_данных);
протокол.проверка_сообщения_по_части_строки("Обработка завершена успешно");
протокол.сохранение(СледующаяФорма.class, null);
```

## Неясности
1. Логика определения класса окна в конструкторе - сложные условия ветвления
2. Назначение метода `searchWindowByAnyClass` - fallback механизм поиска окон
3. Обработка HTML-тегов в сообщениях - почему используется двойная проверка (с тегами и без)
4. Разные задержки в методах сохранения и продолжения (20 и 30 секунд)
