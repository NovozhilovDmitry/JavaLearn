# Диалоговое_окно_с_действием_и_выбором.java

## Описание
Класс для работы с диалоговыми окнами, требующими выбора значения из выпадающего списка перед подтверждением действия. Автоматически находит комбобокс в диалоговом окне и предоставляет методы для выбора значения и подтверждения. Поддерживает различные типы диалогов через перечисление.

## Public методы
- `Диалоговое_окно_с_действием_и_выбором(Имена_Диалоговых_Окон)` - конструктор с типом диалога
- `Диалоговое_окно_с_действием_и_выбором(String)` - конструктор с названием окна
- `выбор(String)` - выбирает значение из выпадающего списка
- `ok(T)`, `ok(Class<T>)`, `ok(Class<T>, Object)` - нажимает кнопку OK с переходом
- `отмена(T)`, `отмена(Class<T>, Object)` - нажимает кнопку отмены
- `проверка_текста_диалогового_окна(String)` - проверяет текст сообщения

## Зависимости
- `forms.general.dialog.abst.DialogWithMessage` - абстрактный базовый класс
- `com.rational.test.ft.object.interfaces.TestObject` - RFT тестовые объекты
- `userInterface.ComboBox`, `userInterface.Jump_Button`, `userInterface.Window` - UI-компоненты

## Пример использования
```java
Диалоговое_окно_с_действием_и_выбором диалог = new Диалоговое_окно_с_действием_и_выбором(
    Диалоговое_окно_с_действием_и_выбором.Имена_Диалоговых_Окон.Выбор_операции);
диалог.выбор("Формировать ПД")
      .ok(СледующаяФорма.class, null);
```

## Неясности
1. Алгоритм поиска комбобокса в диалоге - перебор дочерних элементов для нахождения JLabel
2. Закомментированная логика определения priorLabel - почему была заменена на динамический поиск
3. Обработка различных типов диалогов - как определяется текст для выбора в разных сценариях
4. Разница между `Имена_Диалоговых_Окон.Подтверждение_выбора` и `Подтверждение_выбора2` - дублирование в перечислении
