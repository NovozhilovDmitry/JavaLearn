# Диалоговое_окно_с_действием.java

## Описание
Класс для работы с диалоговыми окнами подтверждения действий в системе. Предоставляет стандартизированные методы для взаимодействия с окнами, требующими подтверждения (Да/Нет, Сохранить/Отмена и др.). Содержит перечисление с предопределенными названиями диалоговых окон и автоматически настраивает кнопки в зависимости от типа окна.

## Public методы
- `Диалоговое_окно_с_действием()` - конструктор по умолчанию для окна "Сохранение"
- `Диалоговое_окно_с_действием(Имена_Диалоговых_Окон)` - конструктор с указанием типа диалога
- `Диалоговое_окно_с_действием(String)` - конструктор с произвольным названием окна
- `да()` - нажимает кнопку подтверждения (Да/Сохранить/Продолжить) и закрывает окно
- `да(T)`, `да(Class<T>)`, `да(Class<T>, Object)` - нажимает кнопку с переходом на указанную форму
- `cancelSaveDialog(T)`, `cancelSaveDialog(Class<T>)`, `cancelSaveDialog(Class<T>, Object)` - нажимает кнопку отмены
- `нажать_Включить_в_расчет(Class<T>, Object)` - нажимает кнопку "Включить в расчет"
- `нажать_продолжить(Class<T>, Object)` - нажимает кнопку "Продолжить"
- `проверка_текста_диалогового_окна(String)` - проверяет текст в диалоговом окне
- `проверка_всего_текста_диалогового_окна(String...)` - проверяет все тексты labels в окне

## Зависимости
- `forms.general.dialog.abst.DialogWithMessage` - абстрактный базовый класс
- `enums.FindBy` - перечисление стратегий поиска
- `userInterface.Jump_Button`, `userInterface.Window` - UI-компоненты

## Пример использования
```java
Диалоговое_окно_с_действием диалог = new Диалоговое_окно_с_действием(
    Диалоговое_окно_с_действием.Имена_Диалоговых_Окон.Сохранение);
диалог.проверка_текста_диалогового_окна("Вы уверены, что хотите сохранить изменения?")
      .да(СледующаяФорма.class);
```

## Неясности
1. Логика определения кнопок в конструкторе - сложное ветвление по названиям окон
2. Разница между методами `да()` с разными параметрами - когда использовать каждый вариант
3. Назначение кнопок "включить_в_расчет" и "продолжить" - в каких конкретно окнах они появляются
4. Устаревший метод `проверка_text_area_диалогового_окна` - почему был заменен
