# Сообщение.java

## Описание
Класс для работы с информационными модальными окнами типа "Сообщение". Предоставляет методы для чтения текста сообщений, проверки содержания и закрытия окон. Поддерживает как стандартные сообщения, так и окна с динамическими заголовками. Использует паттерн Singleton для доступа к текущему экземпляру.

## Public методы
- `Сообщение()` - конструктор по умолчанию для окна "Сообщение"
- `Сообщение(Имена_Диалоговых_Окон)` - конструктор с указанием типа сообщения
- `Сообщение(String)` - конструктор с произвольным названием окна
- `getInstance()` - возвращает экземпляр класса (Singleton)
- `getText()` - возвращает полный текст сообщения
- `getTexts()` - возвращает список текстов по элементам
- `getTextEntries()` - возвращает список записей текста
- `выход_из_диалогового_окна_Сообщение()` - нажимает OK и закрывает окно с переходом
- `проверка_текста_диалогового_окна_Сообщение(String)` - проверяет текст сообщения
- `проверка_текста_ignore_case(String)` - проверяет текст без учета регистра
- `проверка_текста_диалогового_окна_Сообщение(Function)` - проверяет текст через условие

## Зависимости
- `forms.general.dialog.abst.DialogWithMessage` - абстрактный базовый класс
- `enums.FindBy` - перечисление стратегий поиска
- `userInterface.Jump_Button`, `userInterface.Window` - UI-компоненты

## Пример использования
```java
Сообщение сообщение = new Сообщение(Сообщение.Имена_Диалоговых_Окон.Сохранение);
сообщение.проверка_текста_диалогового_окна_Сообщение("Данные успешно сохранены")
         .выход_из_диалогового_окна_Сообщение(СледующаяФорма.class);
```

## Неясности
1. Логика обработки специальных окон ("Выберите подписи для формы 1", "Проверка реквизитов") - особые кнопки
2. Назначение метода `нажать_Продолжить` - не инициализируется в конструкторе
3. Разница между `getText()` и `getTexts()` - когда использовать каждый метод
4. Обработка многострочных сообщений - как именно разделяются тексты по элементам
