# БлокФильтровТаблицы.java

## Описание
Интерфейс для работы с блоками фильтров в табличных формах. Предоставляет методы для наложения, проверки и отмены фильтров через построитель фильтров. Поддерживает сложные сценарии фильтрации по нескольким значениям и автоматизирует взаимодействие с конструктором выражений.

## Public методы
- `кнопкаЗадатьФильтр()`, `кнопкаОтменитьФильтр()` - возвращают кнопки фильтров
- `задать_фильтр()` - открывает построитель фильтров
- `проверить_фильтр()` - проверяет значения фильтра
- `проверка_доступности_кнопки()` - проверяет доступность кнопок
- `наложить_фильтр()` - накладывает фильтр по одному полю
- `отменить_фильтр()` - отменяет активный фильтр
- `наложить_фильтр_по_нескольким_значениям_поля_Искать_среди_них()` - сложная фильтрация по нескольким значениям

## Зависимости
- `forms.general.Построитель_фильтра` - построитель фильтров
- `forms.general.abstractForms.FormUnit` - родительский интерфейс
- `enums.FindBy` - перечисление стратегий поиска
- `userInterface.Button`, `userInterface.Jump_Button` - UI-компоненты

## Пример использования
```java
public class ФормаСФильтрами implements БлокФильтровТаблицы<ФормаСФильтрами> {
    // реализация FormWin
    public void применитьФильтр() {
        наложить_фильтр("Дата", "Равно", "01.01.2024");
    }
}
```

## Неясности
1. Алгоритм формирования сложных фильтров в `наложить_фильтр_по_нескольким_значениям` - логика сборки выражения
2. Назначение задержки в `отменить_фильтр_и_проверить_отмена_недоступна` - почему именно 2 секунды
3. Обработка ошибок при наложении фильтров - какие исключения могут возникнуть
