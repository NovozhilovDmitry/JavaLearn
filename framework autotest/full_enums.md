# FindBy.java

## Summary
Комплексный enum для стратегий поиска UI-элементов в Swing-приложениях. Содержит основные стратегии поиска и вложенные enum'ы для конкретных типов компонентов. Используется для локации элементов по классу, тексту, имени и другим атрибутам.

## Public методы
1. **getValue()**  
   Возвращает строковое значение стратегии поиска. Не имеет параметров.

## Вложенные enum'ы и интерфейсы
- **ConstantClass**: интерфейс для enum'ов классов компонентов
- **ButtonClass, LabelClass, ComboBoxClass, ListClass, TableClass, TextFieldClass, CheckBox_RadBtn_Class, TabClass, TabPanelClass, ProgressBarClass**: enum'ы конкретных UI-компонентов
- **TextValue**: enum с русскоязычными текстами для поиска

## Dependencies
Нет внешних зависимостей.

## Usage Example
```java
// Поиск по классу
FindBy.ButtonClass buttonClass = FindBy.ButtonClass.AsdcoJButton;

// Поиск по тексту
FindBy.TextValue textValue = FindBy.TextValue.Да;

// Использование в локаторе
String locator = buttonClass.getcls() + "=" + buttonClass.getValue();
```

## Uncertainties
1. Некоторые значения в enum'ах могут быть устаревшими или неиспользуемыми
2. Наличие закомментированного поля ComboBoxClass может указывать на недоделанную функциональность
3. Неясно, как именно используются эти константы в фреймворке автоматизации
4. Некоторые имена классов выглядят сгенерированными (FrmGate$7, FrmNsi$26)

# Projects.java

## Summary
Enum, представляющий различные проекты в системе. Содержит информацию о названии проекта, соответствующей схеме базы данных и параметрах системных настроек. Используется для управления конфигурациями проектов, включая определение шлюзов и параметров подключения к БД.

## Public методы
1. **hasItsOwnGates()**  
   Проверяет, имеет ли проект собственные шлюзы. Возвращает boolean.

2. **getByValue(String value)**  
   Статический метод для получения экземпляра Projects по строковому значению. Параметр: value - строковое значение проекта. Может выбросить AssertionError если проект не найден.

3. **currentProject()**  
   Статический метод, возвращающий текущий проект на основе MainClass.currentProject.

## Dependencies
- base.MainClass (для currentProject)
- java.util.Arrays, Collections, HashMap, Map

## Usage Example
```java
// Получение текущего проекта
Projects current = Projects.currentProject();

// Проверка наличия шлюзов
boolean hasGates = Projects.DEPOSIT.hasItsOwnGates();

// Получение проекта по значению
Projects credit = Projects.getByValue("credit");
```

## Uncertainties
1. Назначение nv_parameterName не до конца ясно из комментария
2. Логика hasItsOwnGates() (value.equals(schema)) может быть неочевидной
3. Неясно, как именно используется DB_SCHEMAS в других частях системы

# XmlTemplate.java

## Summary
Интерфейс для определения контракта XML-шаблонов. Задает основные атрибуты XML-документов: версию, кодировку, standalone-статус и имя схемы. Используется для стандартизации создания и валидации XML-документов в системе.

## Public методы
1. **getVersion()**  
   Возвращает версию XML-документа (например, "1.0").

2. **getEncoding()**  
   Возвращает кодировку XML-документа.

3. **getStandalone()**  
   Возвращает standalone-статус документа (yes/no).

4. **getSchema_name()**  
   Возвращает имя XML-схемы (XSD) для валидации.

## Dependencies
Нет внешних зависимостей.

## Usage Example
```java
// Реализация интерфейса
public class CreditTemplate implements XmlTemplate {
    public String getVersion() { return "1.0"; }
    public String getEncoding() { return "UTF-8"; }
    public Boolean getStandalone() { return true; }
    public String getSchema_name() { return "credit.xsd"; }
}
```

## Uncertainties
1. Неясно, как именно используется этот интерфейс в фреймворке
2. Отсутствуют комментарии о допустимых значениях для каждого метода
3. Не определены ожидаемые форматы для версий и имен схем

# Кодировки.java

## Summary
Простой enum для представления текстовых кодировок, используемых в системе. Содержит две основные кодировки: Windows-1251 (кириллическая) и UTF-8 (универсальная). Используется для указания кодировки при работе с файлами, XML-документами и другими текстовыми ресурсами.

## Public методы
1. **getValue()**  
   Возвращает строковое представление кодировки. Не имеет параметров.

## Dependencies
Нет внешних зависимостей.

## Usage Example
```java
// Получение значения кодировки
String encoding = Кодировки.UTF_8.getValue();

// Использование при чтении файлов
FileReader reader = new FileReader(file, Кодировки.Windows_1251.getValue());
```

## Uncertainties
1. Отсутствуют комментарии о конкретных сценариях использования каждой кодировки
2. Нет методов для проверки поддержки кодировки или преобразования между ними

# ОперационныеДни.java

## Summary
Enum, определяющий различные типы операционных дней, используемые в банковской системе. Включает текущий операционный день, следующие дни, календарные и рабочие дни с различными смещениями. Используется для вычисления дат в финансовых операциях и отчетностях.

## Public методы
Нет public методов, только значения перечисления.

## Dependencies
Нет внешних зависимостей.

## Usage Example
```java
// Использование в SQL-запросах
String query = "SELECT * FROM transactions WHERE date = " + ОперационныеДни.ОПЕРАЦИОННЫЙ_ДЕНЬ;

// Использование в бизнес-логике
if (dayType == ОперационныеДни.СЛЕДУЮЩИЙ_ОПЕРАЦИОННЫЙ_ДЕНЬ) {
    // обработка следующего операционного дня
}
```

## Uncertainties
1. Неясна точная разница между "кд" (календарные дни) и "рд" (рабочие дни)
2. Назначение СРОК_ОВН не расшифровано
3. Отсутствуют комментарии о бизнес-логике каждого типа дня
4. Нет методов для вычисления фактических дат на основе этих типов

// Использование в тестах авторизации

# Расширения_файлов.java

## Summary
Enum для работы с файловыми расширениями в системе. Содержит основные расширения файлов, используемые в банковской системе, включая готовые (.ready) и архивные форматы. Предоставляет утилитные методы для определения расширения файла и проверки его наличия.

## Public методы
1. **getРасширение()**  
   Возвращает строковое значение расширения файла.

2. **haveExtensionOf(File file)**  
   Проверяет, имеет ли файл одно из известных расширений. Параметр: file - проверяемый файл.

3. **haveExtensionOf(String fName)**  
   Проверяет, имеет ли имя файла одно из известных расширений. Параметр: fName - имя файла.

4. **from(String fName)**  
   Определяет расширение файла по его имени. Возвращает Optional<Расширения_файлов>.

## Dependencies
- java.io.File
- java.util.Optional

## Usage Example
```java
// Проверка расширения файла
boolean isExcel = Расширения_файлов.haveExtensionOf("report.xls");

// Получение расширения
Optional<Расширения_файлов> ext = Расширения_файлов.from("data.xml");
if (ext.isPresent()) {
    // обработка XML файла
}

// Использование в фильтрации
File file = new File("document.txt");
if (Расширения_файлов.haveExtensionOf(file)) {
    // обработка текстового файла
}
```

## Uncertainties
1. Логика обработки расширения "empty" может быть неочевидной
2. Не все возможные расширения файлов покрыты enum'ом
3. Метод from() не обрабатывает файлы с несколькими точками в имени

# Шлюзы.java

## Summary
Комплексный enum, представляющий шлюзы (gateways) банковской системы для обмена файлами. Содержит как технические шлюзы (amhc, arc, dmcp), так и территориальные отделения банка России. Управляет путями для входящих и исходящих файлов, обеспечивает интеграцию между различными подсистемами.

## Public методы
1. **непустыеИсхШлюзы()**  
   Статический метод, возвращающий список непустых исходящих шлюзов. Возвращает List<Gate>.

2. **getNamePart()**  
   Возвращает часть пути шлюза. Для шлюза tmp переопределен для чтения из свойств.

3. **getВходящий()**  
   Возвращает путь входящего шлюза. Может выбросить AssertionError если шлюз не найден.

4. **getИсходящий()**  
   Возвращает путь исходящего шлюза. Может выбросить AssertionError если шлюз не найден.

5. **asGate()**  
   Преобразует шлюз в объект Gate.

## Dependencies
- base.MainClass (для targetProject)
- bugbusters.modules.helper.utils.ProjectProperties, StringUtils
- db.sql.MainSql (для инициализации шлюзов)
- model.Gate
- Environment.config_enviroment
- JUnit 5 (Assertions.fail)

## Usage Example
```java
// Получение пути шлюза
String incomingPath = Шлюзы.ODBR.getВходящий();
String outgoingPath = Шлюзы.ODBR.getИсходящий();

// Работа с непустыми шлюзами
List<Gate> activeGates = Шлюзы.непустыеИсхШлюзы();

// Преобразование в Gate объект
Gate gate = Шлюзы.KO.asGate();
```

## Uncertainties
1. Назначение многих шлюзов не документировано (amhc, dvos, dmcp и др.)
2. Логика инициализации вхШлюзы и исхШлюзы зависит от MainSql
3. Шлюз tmp имеет особую логику работы
4. Закомментированные территориальные отделения в конце enum'а
5. Потенциальные проблемы с путями (комментарий про добавление "\\")
6. Не ясна разница между ODBR и ODBRin
