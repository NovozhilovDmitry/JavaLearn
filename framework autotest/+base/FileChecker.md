# FileChecker.java - Документация

## Summary
Интерфейс `FileChecker` предоставляет базовую функциональность для работы с файлами в тестовом фреймворке. Он включает методы для обработки ошибок, выгрузки файлов в шлюзы и работы с содержимым файлов. Интерфейс интегрирован с системой отчетов Allure для прикрепления файлов и логирования шагов тестирования.

## Public методы
- **`writeException(Throwable e)`** - сохраняет исключение в файл и прикрепляет к отчету Allure (активируется через системное свойство `FilesChecker.safetyCheck`)
- **`выгрузить_в_Шлюз(Шлюзы шлюз)`** - выгружает файл в указанный шлюз с копированием через временную директорию
- **`файл()`** - абстрактный метод, возвращающий объект File для проверки
- **`bytes()`** - возвращает содержимое файла в виде byte[] (бросает assertion error при ошибке чтения)

## Dependencies
- **`bugbusters.modules.extensions.allure.*`** - интеграция с Allure для шагов и вложений
- **`enums.Шлюзы`** - перечисление шлюзов для выгрузки файлов
- **`bugbusters.modules.helper.utils.DateGenerator`** - утилиты для работы с датами
- **Внешние**: `org.slf4j.Logger`, `java.nio.file.*`, `org.junit.jupiter.api.Assertions`

## Usage Example
```java
public class MyFileChecker implements FileChecker {
    @Override
    public File файл() {
        return new File("test.xml");
    }
    
    public void processFile() {
        выгрузить_в_Шлюз(Шлюзы.ШЛЮЗ_ПИС);
        byte[] content = bytes();
    }
}
```

## Uncertainties
1. **Условие безопасности** - логика активации через `FilesChecker.safetyCheck` может быть неочевидной
2. **Обработка ошибок** - метод `bytes()` завершает тест при ошибке, что может быть избыточно
3. **Временные файлы** - стратегия очистки временных файлов после выгрузки не определена
4. **Кодировка файлов** - не указана кодировка при записи исключений
