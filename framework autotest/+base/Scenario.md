# Scenario.java - Документация

## Summary
Абстрактный класс `Scenario` является базовым классом для тестовых сценариев, предоставляющим общую инфраструктуру для инициализации тестового окружения. Он автоматически определяет проект по имени класса, инициализирует главный класс АРМ и управляет каталогами тестовых данных. Класс реализует шаблон сценария с методами подготовки и очистки.

## Public методы
- **`createMainClass(Projects project)`** - фабричный метод для создания экземпляра MainClass (можно переопределить)
- **`beforeMainTest()`** - инициализирует тестовое окружение на основе имени класса
- **`beforeScenario()`** - абстрактный метод для подготовки конкретного сценария
- **`считывание_эталонных_файлов()`** - метод для загрузки эталонных файлов (можно переопределить)
- **`каталог_тестовых_данных()`** - возвращает путь к каталогу тестовых данных текущего теста
- **`эталонныйФайл(String... subDirsAndFileName)`** - статический метод для получения пути к эталонному файлу

## Dependencies
- **`base.Base`** - базовый класс тестового фреймворка
- **`base.AbstractTest`** - интерфейс для работы с тестовыми данными
- **`enums.Projects`** - перечисление проектов
- **`io.qameta.allure.Step`** - интеграция с Allure для шагов тестирования
- **Внешние**: `org.slf4j.Logger`, `java.nio.file.*`

## Usage Example
```java
public class PaymentScenario extends Scenario {
    @Override
    protected void beforeScenario() {
        // Подготовка данных для теста
    }
    
    @Test
    public void testPayment() {
        // Тестовый сценарий
    }
}
```

## Uncertainties
1. **Определение проекта** - логика определения проекта по имени класса может быть ненадежной
2. **Статическое состояние** - поле `prevTestClass` может вызвать проблемы в параллельных тестах
3. **Наследование** - множественное наследование (Base + AbstractTest) усложняет архитектуру
