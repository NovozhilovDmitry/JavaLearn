# EnvelopePIS - Документация

## Summary
Класс `EnvelopePIS` представляет интеграционный конверт ПИС 1Х в формате ZIP-архива (GUID1.zip). Он обеспечивает работу с вложенными компонентами конверта: метаинформацией, электронными документами, квитанциями обработки и статусами. Класс поддерживает как входящие, так и исходящие сообщения и предоставляет инструменты для валидации XML-структур по заданным шаблонам.

## Public методы

### Статические методы построителя
- **`referenceBuilder(String имяЭталонногоФайла)`** - создает Builder для поиска эталонного файла в тестовых данных
- **`builder(String fileNamePart, Шлюзы... шлюзы)`** - создает Builder для поиска файла в шлюзах по имени/части имени
- **`builder(Path path)`** - создает Builder по конкретному пути к файлу

### Методы экземпляра
- **`getArchive()`** - возвращает основной ZIP-архив конверта (GUID1.zip)
- **`yMetaJson()`** - возвращает метаинформацию конверта из y.meta.json
- **`getDocument()`** - возвращает вложенный документ (GUID2.zip) с электронным сообщением
- **`getESODReciept()`** - возвращает результат обработки ESOD (GUID3)
- **`getStatus()`** - возвращает параметры статуса обработки для ЛК ЕПВВ
- **`getSign()` / `setSign(String)`** - геттер/сеттер для УКЭП Абонента

## Dependencies
**Внутренние утилиты:**
- `ZIP_archive` - работа с ZIP-архивами
- `FileSearcher` - поиск файлов
- `XmlFileChecker` - проверка XML файлов
- `YMetaJson` - парсинг метаинформации

**Внешние зависимости:**
- `com.google.gson.Gson` - JSON сериализация
- `org.opentest4j.AssertionFailedError` - обработка ошибок тестирования

## Usage Example
```java
// Создание конверта из файла в шлюзе
EnvelopePIS envelope = EnvelopePIS.builder("GUID1", Шлюзы.ШЛЮЗ_ПИС)
    .withEd(XmlTemplate.ED_ALBUM)
    .withEsod(XmlTemplate.ESOD_ALBUM)
    .build();

// Работа с компонентами
EDDocument document = envelope.getDocument();
XmlFileChecker xml = document.getXml();
Status status = envelope.getStatus();
```

## Uncertainties
1. **Логика определения типа сообщения** - как именно определяется входящее/исходящее сообщение через `yMetaJson.isIncoming()`
2. **Структура YMetaJson** - точный формат и обязательные поля в y.meta.json
3. **Обработка ошибок архива** - неясна стратегия обработки поврежденных архивов
4. **Многопоточная безопасность** - использование класса в параллельных тестах
5. **Временные файлы** - стратегия очистки временных файлов после извлечения
6. **Валидация подписи** - отсутствует явная валидация УКЭП
