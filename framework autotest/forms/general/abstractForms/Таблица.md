# Таблица.java

## Описание
Интерфейс для работы с табличными данными в формах. Предоставляет комплексный набор методов для взаимодействия с таблицами: поиск строк, проверка значений, сортировка, получение данных. Использует кэширование для оптимизации повторного доступа к таблицам и поддерживает работу с различными типами данных.

## Public методы
- `таблица()` - возвращает объект таблицы (с кэшированием)
- `getRowCount()`, `getColumnCount()` - возвращают размеры таблицы
- `выделение_строки()` - выделяет строку по значениям колонок
- `получение_строки()` - получает данные строки
- `проверка_отсутствия_строки_таблицы()` - проверяет отсутствие строки
- `получить_значения_Колонки()` - получает значения колонки
- `колонка_отсортирована()` - проверяет сортировку колонки
- `сумма_Значений_Колонки()` - вычисляет сумму значений колонки
- `получение_таблицы()` - получает всю таблицу как список мап

## Зависимости
- `enums.FindBy`, `enums.TableColumn` - перечисления для поиска и колонок
- `userInterface.Table` - UI-компонент таблицы
- `forms.general.abstractForms.FormUnit` - родительский интерфейс

## Пример использования
```java
public class ФормаСТаблицей implements Таблица<ФормаСТаблицей> {
    // реализация FormWin
    public void работатьСТаблицей() {
        выделение_строки(new RowArgs("Колонка", "Значение"));
        String[] значения = получить_значения_Колонки("ИмяКолонки");
    }
}
```

## Неясности
1. Назначение кэша `LAST_TABLES` - логика очистки и управления временем жизни
2. Разница между `RowArgs` и `TableColumn` - когда использовать каждый подход
3. Логика работы `уникальная_колонка()` - алгоритм определения уникальности
