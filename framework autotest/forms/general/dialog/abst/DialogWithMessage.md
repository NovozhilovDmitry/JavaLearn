# DialogWithMessage.java

## Описание
Абстрактный базовый класс для диалоговых окон, содержащих текстовые сообщения. Предоставляет общую функциональность для чтения и проверки текста в модальных окнах. Использует алгоритмы поиска подстрок с учетом различных форматов пробелов и регистра, а также обеспечивает сортировку текстовых элементов по их расположению на экране.

## Public методы
- `проверка_текста_диалогового_окна(String expText, boolean ignoreCase)` - проверяет текст в диалоговом окне с учетом регистра и пробелов
- `readMessage()` - считывает все текстовые сообщения из окна и объединяет их в строку
- `readMessage(Window win)` - считывает все текстовые элементы из указанного окна как DOM-дерево

## Вложенный класс Entry
- `Entry(TestObject ref)` - конструктор для создания записи из тестового объекта
- `getCls()` - возвращает класс элемента
- `getText()` - возвращает текст элемента
- `getLocationOnScreen()` - возвращает расположение элемента на экране
- `toString()` - строковое представление элемента

## Зависимости
- `bugbusters.modules.helper.utils.StringUtils` - утилиты для работы со строками
- `com.rational.test.ft.object.interfaces.TestObject` - RFT тестовые объекты
- `enums.FindBy` - перечисление стратегий поиска
- `forms.general.abstractForms.FormWin` - интерфейс оконных форм
- `userInterface.utils.TestObjectFinder` - утилиты поиска тестовых объектов
- `utils.SubstringSearcher` - поиск подстрок

## Пример использования
```java
// В наследующем классе
public class КонкретноеДиалоговоеОкно extends DialogWithMessage<КонкретноеДиалоговоеОкно> {
    // реализация методов
}

КонкретноеДиалоговоеОкно диалог = new КонкретноеДиалоговоеОкно();
диалог.проверка_текста_диалогового_окна("Ошибка сохранения", false);
String текстСообщения = диалог.readMessage();
```

## Неясности
1. Назначение двойной проверки в `проверка_текста_диалогового_окна` - сначала с пробелами, потом без
2. Алгоритм фильтрации элементов по классам из `FindBy.LabelClass` - какие именно классы считаются текстовыми
3. Логика сортировки элементов по координате Y - как это влияет на порядок чтения текста
4. Использование дженерика `T extends DialogWithMessage<T>` - паттерн CRTP и его назначение
