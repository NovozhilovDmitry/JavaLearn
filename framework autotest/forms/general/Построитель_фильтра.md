# Построитель_фильтра.java

## Описание
Класс для работы с конструктором фильтров в GUI-приложении. Предоставляет функциональность для создания и проверки фильтров через конструктор выражений. Поддерживает различные типы ввода данных и обработку ошибок через модальные сообщения.

## Public методы
- `проверить_значения_фильтра(String...)` - проверяет значения в строке таблицы фильтров
- `проверкаЗначенияСтолбца(String, String)` - проверяет значение в конкретном столбце
- `выбор_поля(String)` - выбирает поле с переходом в конструктор выражений
- `выбор_поля_без_выражения(String)` - выбирает поле без конструктора выражений
- `применить_фильтр(String, String, String)` - применяет фильтр с параметрами
- `проверить_сформированное_значение(String)` - проверяет итоговое значение фильтра
- `применить_фильтр(Class<T>, Object)` и `применить_фильтр(T)` - применяют фильтр с переходом

## Зависимости
- `base.PropertyBuilder` - построитель свойств для поиска объектов
- `forms.general.dialog.Сообщение` - класс для работы с диалоговыми окнами
- `userInterface.*` - UI-компоненты
- `userInterface.utils.TestObjectFinder` - утилиты поиска тестовых объектов

## Пример использования
```java
Построитель_фильтра построитель = new Построитель_фильтра();
построитель.выбор_поля("Дата")
    .применить_выражение("Равно", "01.01.2024")
    .применить_фильтр(НоваяФорма.class, null);
```

## Неясности
1. Логика определения `typeToSetValue` - эвристика на основе наличия "лайков"
2. Назначение системного свойства `ExpressionConstructor.checkIncompletedParameterMessage.millis`
3. Алгоритм повторной попытки ввода значения при появлении сообщения
4. Разница между методами `применить_фильтр` с разными параметрами
