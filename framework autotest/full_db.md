# JDBCConnect.java

## Summary
Класс для управления соединениями с Oracle базой данных. Реализует пул соединений с синхронизацией для многопоточного доступа. Автоматически загружает драйвер Oracle и настраивает параметры соединения из свойств окружения. Обеспечивает единственное соединение на схему.

## Public методы
1. **get()**  
   Статический метод для получения соединения из пула. Возвращает Connection для текущего проекта.

2. **JDBCConnect()**  
   Конструктор по умолчанию - создает новое соединение.

3. **JDBCConnect(Connection existingConnection)**  
   Конструктор с существующим соединением.

4. **getConnection()**  
   Возвращает текущее соединение (не рекомендуется для прямого использования).

## Dependencies
- Oracle JDBC Driver (oracle.jdbc.driver.OracleDriver)
- Projects enum и Projects.DB_SCHEMAS для маппинга схем
- Environment.getDbProperties для параметров подключения
- JUnit 5 (Assertions.fail) для обработки ошибок

## Usage Example
```java
// Получение соединения через пул
Connection conn = JDBCConnect.get();

// Использование в Table классе
Statement stmt = JDBCConnect.get().createStatement();
```

## Uncertainties
1. Пул соединений не ограничен по размеру
2. Нет механизма проверки валидности соединений в пуле
3. Закрытие старых соединений в отдельном потоке может вызвать race conditions
4. Не обрабатываются случаи, когда соединение становится невалидным
5. AutoCommit отключен, что может привести к накоплению транзакций
6. Схема определяется через currentProject, что может быть небезопасно в многопоточности

# Points.java

## Summary
Класс для работы с точками восстановления (restore points) в Oracle БД. Позволяет создавать, восстанавливать и удалять точки восстановления для тестовых целей. Автоматически генерирует имена точек по шаблону и управляет их жизненным циклом. Особенно полезен для отката состояния базы данных после выполнения тестов.

## Public методы
1. **создание_короткой_точки(String название_съюта)**  
   Создает точку восстановления с автоматически сгенерированным именем. Параметр: название_съюта - префикс для имени точки.

2. **восстановление_короткой_точки(@Nullable String часть_имени_точки)**  
   Восстанавливает состояние базы до указанной точки. Параметр: часть_имени_точки - если null, восстанавливает последнюю точку.

3. **восстановление_последней_короткой_точки()**  
   Восстанавливает последнюю созданную точку без указания имени.

4. **удаление_точек(String часть_названия, List<String> имена_точек)**  
   Удаляет точки по части имени или списку имен. Параметры: часть_названия - фильтр по имени, имена_точек - конкретные точки для удаления.

## Dependencies
- Oracle Database (v$restore_point, FLASHBACK TABLE)
- Внутренние утилиты: DateGenerator для работы с датами
- Projects enum (CREDIT, DEPOSIT)
- Table.execute, Table.getTable, Table.executeQueryData для SQL-запросов
- Environment.getDbProperties для получения параметров БД

## Usage Example
```java
// Создание точки восстановления
Points.создание_короткой_точки("MyTestSuite");

// Восстановление до конкретной точки
Points.восстановление_короткой_точки("MyTestSuite");

// Восстановление последней точки
Points.восстановление_последней_короткой_точки();
```

## Uncertainties
1. Логика определения "working" проектов закомментирована и не функционирует
2. Использование кириллических идентификаторов может вызвать проблемы с кодировкой
3. Ограничение в 1000 точек (1-999) может быть недостаточным
4. Нет обработки случаев, когда точка восстановления уже существует
5. Потенциальные проблемы с правами доступа к v$restore_point
6. Не ясна логика работы с временными метками в удалении устаревших точек

# StatementChecker.java

## Summary
Утилитный класс для проверок существования записей в базе данных через SQL-запросы. Предоставляет статические методы для проверки наличия, отсутствия и обновления записей. Все методы включают Allure-степы для отчетности и используют JUnit assertions для провала тестов.

## Public методы
1. **checkRecordExists(String query)**  
   Проверяет существование хотя бы одной записи по запросу. Параметр: query - SQL запрос, возвращающий количество записей.

2. **checkRecordExists(String query, String msg)**  
   Аналогично предыдущему, но с кастомным сообщением об ошибке.

3. **checkNoRecord(String query, String msg)**  
   Проверяет отсутствие записей по запросу с кастомным сообщением.

4. **checkNoRecord(String query)**  
   Проверяет отсутствие записей по запросу.

5. **checkRecordUpdated(String query)**  
   Проверяет успешность обновления записей.

6. **executeSql(String query)**  
   Выполняет SQL запрос и возвращает результат как строку.

## Dependencies
- JUnit 5 (Assertions.fail)
- Allure Framework (@Step)
- Table.executeQuery для выполнения SQL-запросов
- Environment.getTimeOutProperties для таймаутов

## Usage Example
```java
StatementChecker.checkRecordExists(
    "SELECT COUNT(*) FROM users WHERE status = 'active'"
);

StatementChecker.checkNoRecord(
    "SELECT COUNT(*) FROM temp_data", 
    "Временные данные должны быть очищены"
);
```

## Uncertainties
1. Метод executeSql_rep() объявлен но не реализован
2. Нет валидации SQL-запросов на безопасность
3. Предполагается, что все запросы возвращают числовой результат
4. Закомментированный код содержит альтернативную реализацию
5. Не обрабатываются случаи, когда запрос возвращает NULL

# Table.java

## Summary
Класс-обертка для выполнения SQL-запросов к базе данных. Предоставляет синхронизированные методы для различных типов запросов: SELECT, UPDATE, INSERT, DELETE. Поддерживает возврат результатов в различных форматах: одиночные значения, списки, таблицы. Включает подробное логирование всех операций.

## Public методы
1. **execute(String query, String columName)**  
   Выполняет запрос и возвращает ResultSet. Параметры: query - SQL запрос, columName - имя колонки.

2. **executeQuery(String query, String columName)**  
   Выполняет SELECT и возвращает значение указанной колонки из первой строки.

3. **executeQuery(String query, int columnIndex)**  
   Аналогично предыдущему, но с указанием индекса колонки.

4. **executeQueryData(String query, int row)**  
   Возвращает все колонки указанной строки в виде списка.

5. **executeQueryData(String query, String columnName)**  
   Возвращает все значения указанной колонки в виде списка.

6. **getTable(String query)**  
   Возвращает всю результирующую таблицу как List<LinkedHashMap>.

7. **execute(String sql)**  
   Выполняет любой SQL запрос, возвращает boolean результат.

8. **update_DB(String sql)**  
   Выполняет UPDATE/INSERT/DELETE запросы с коммитом.

## Dependencies
- JDBC (ResultSet, Statement, SQLException)
- Внутренняя утилита StringUtils для форматирования
- RFT_Table.таблица_toString для преобразования таблиц
- JDBCConnect для получения соединения с БД

## Usage Example
```java
// Получение одного значения
String userName = Table.executeQuery(
    "SELECT name FROM users WHERE id = 1", 
    "name"
);

// Получение списка значений
List<String> userNames = Table.executeQueryData(
    "SELECT name FROM users", 
    "name"
);

// Выполнение UPDATE
Table.update_DB("UPDATE users SET status = 'inactive'");
```

## Uncertainties
1. Два deprecated метода (executeStatement) не синхронизированы
2. Метод executeQueryData(String, int) может быть неочевидным в использовании
3. Нет обработки случаев, когда запрос возвращает пустой результат
4. Логирование может раскрывать чувствительные данные
5. Не ясна логика работы с большими объемами данных

# MainSql.java

## Summary
Класс для инициализации системных данных через SQL-запросы. Собирает информацию о шлюзах и операционных днях для различных проектов системы. Использует многопоточность для параллельного выполнения SQL-запросов к базе данных. Автоматически определяет пути для входящих/исходящих шлюзов и даты различных типов операционных дней.

## Public методы
1. **MainSql()**  
   Конструктор - основной метод класса. Выполняет инициализацию данных для всех проектов, которые имеют собственные шлюзы. Запускает параллельные задачи для сбора информации о шлюзах и операционных днях.

2. **readAppVersions(Projects prj)**  
   Статический метод для чтения версии приложения из базы данных. Параметр: prj - проект, для которого запрашивается версия. Сохраняет результат в статическую мапу App.VERSIONS.

## Dependencies
- База данных (SQL-запросы через StatementChecker.executeSql и Table.executeQuery)
- Внутренние enum: Projects, ОперационныеДни, Шлюзы
- Многопоточность: ConcurrentUtils для параллельного выполнения задач
- GatesCollector - сборщик информации о шлюзах
- StringUtils - утилиты для работы со строками
- Environment.getClientProperties - доступ к настройкам клиента

## Usage Example
```java
// Автоматическая инициализация всех шлюзов и операционных дней
new MainSql();

// Получение версии приложения для конкретного проекта
MainSql.readAppVersions(Projects.CREDIT);
```

## Uncertainties
1. Назначение и структура карт ШЛЮЗЫ_OUT, ШЛЮЗЫ_IN и constants (объявлены в родительском классе)
2. Логика определения "hasItsOwnGates()" для проектов
3. Назначение константы "tmp" и ее использование в gatesCollector.put()
4. Формат и структура данных в таблице GATE_HIST и calendar
5. Смысл различных типов операционных дней (60_кд, 61_кд, 6_рд, 7_рд, 1_кг)
6. Обработка ошибок при выполнении SQL-запросов не предусмотрена
7. Потенциальные проблемы с многопоточным доступом к разделяемым ресурсам
