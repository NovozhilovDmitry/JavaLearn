# Форма_выбора_подписей.java

## Описание
Класс для работы с формой выбора подписей для различных документов. Содержит выпадающие списки для выбора ФИО формирующего, подтверждающего и уполномоченного лица, а также кнопку перехода далее. Поддерживает проверку заполнения списков и выбор стандартных подписей.

## Public методы
- `Форма_выбора_подписей(Имена_Окон)` - конструктор с указанием типа формы
- `getInstance()` - возвращает экземпляр класса (Singleton)
- `проверка_списков()` - проверяет что все выпадающие списки не пустые
- `проверить_поля(String, String, String)` - проверяет тексты меток полей
- `выбор_подписывающих(String, String, String)` - выбирает подписывающих из списков
- `выбор_подписывающих()` - выбирает подписывающих по умолчанию
- `далее()` - нажимает кнопку "Далее" с возвратом следующей формы

## Зависимости
- `forms.general.abstractForms.FormWin` - интерфейс оконных форм
- `bugbusters.modules.helper.Helper` - утилиты с задержками
- `org.junit.jupiter.api.Assertions` - assertions для проверок

## Пример использования
```java
Форма_выбора_подписей форма = new Форма_выбора_подписей(
    Форма_выбора_подписей.Имена_Окон.Выберите_подписи_для_формы_1);
форма.выбор_подписывающих("Панкова Н.Б.", "Иванова Н.Ю.", "Иванова Н.Ю.");
форма.далее(СледующаяФорма.class);
```

## Неясности
1. Назначение статических полей `фио1_формирующего` и других - стандартные значения подписей
2. Большая задержка в конструкторе (40 минут) - почему такой длинный таймаут
3. Логика метода `tryDuring` в `выбор_подписывающих` - обработка возможных проблем при выборе
4. Разные варианты методов `далее` - перегрузки для разных сценариев перехода

# Построитель_фильтра.java

## Описание

## Public методы
- `проверить_значения_фильтра(String...)` - проверяет значения в строке таблицы фильтров
- `проверкаЗначенияСтолбца(String, String)` - проверяет значение в конкретном столбце
- `выбор_поля(String)` - выбирает поле с переходом в конструктор выражений
- `выбор_поля_без_выражения(String)` - выбирает поле без конструктора выражений
- `применить_фильтр(String, String, String)` - применяет фильтр с параметрами
- `проверить_сформированное_значение(String)` - проверяет итоговое значение фильтра
- `применить_фильтр(Class<T>, Object)` и `применить_фильтр(T)` - применяют фильтр с переходом

## Зависимости

- `forms.general.dialog.Сообщение` - класс для работы с диалоговыми окнами

## Пример использования
```java

построитель.выбор_поля("Дата")
```

## RowArgs.java

### Summary
Класс `RowArgs` представляет собой контейнер для хранения пар "колонка-значение" в виде массива строк. Используется для удобного построения аргументов строки данных в таблицах. Поддерживает два типа колонок: строковые и перечисляемые (`TableColumn`). Предоставляет методы для преобразования в словарь, добавления новых пар и модификации существующих значений.

### Public Methods
- **`args(): String[]`**  
  Возвращает внутренний массив аргументов в формате `[колонка1, значение1, колонка2, значение2, ...]`.

- **`args_AsLinkedHashMap(): LinkedHashMap<String, String>`**  
  Преобразует массив аргументов в `LinkedHashMap`, где ключи — названия колонок, значения — соответствующие им значения. Сохраняет порядок добавления.

- **`add(RowArgs row_args): RowArgs`**  
  Добавляет аргументы из другого экземпляра `RowArgs`. Если колонка уже существует — обновляет её значение, иначе добавляет новую пару в начало массива. Возвращает текущий объект.

- **`replaceIfPresent(String columnName, UnaryOperator<String> valueReplacer): void`**  
  Заменяет значение для указанной колонки, если она присутствует, используя функцию-модификатор `UnaryOperator`.

- **Статические методы `pairs(...)`**  
  Фабричные методы для создания экземпляров с заданными парами колонка-значение. Поддерживают от 1 до 20 пар. Перегружены для работы со строками и `TableColumn`.

### Dependencies
- `enums.TableColumn` — перечисление для типизированных колонок.
- `java.util.Arrays` — для работы с массивами.
- `java.util.LinkedHashMap` — для возврата упорядоченного словаря.
- `java.util.function.UnaryOperator` — для функциональной замены значений.
- `java.util.stream.Stream` — для операций с массивами.

### Usage Example
```java
RowArgs row = RowArgs.pairs(
    TableColumn.NAME, "Иван",
    TableColumn.AGE, "30"
);
row.add(RowArgs.pairs("Город", "Москва"));
row.replaceIfPresent("Возраст", old -> old + " лет");
LinkedHashMap<String, String> map = row.args_AsLinkedHashMap();
```

### Uncertainties
1. **Логика добавления колонок в `add`**: Новые пары добавляются в начало массива, что может нарушить исходный порядок.
2. **Использование `Arrays.binarySearch`**: Массив не отсортирован, поэтому результат поиска может быть непредсказуемым.
3. **Обработка дубликатов**: Неясно, как обрабатываются колонки с одинаковыми именами при создании через `pairs`.
4. **Производительность**: Частое изменение массива через `Stream.concat` может быть затратным при большом количестве пар.
5. **Кириллические имена**: Использование русских названий параметров может вызвать проблемы с кодировкой в некоторых средах.
